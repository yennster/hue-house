# Hue House for iOS

iOS app target that ships the same Hue networking, gradient, and color logic as the macOS app — packaged for iPhone and iPad. The Xcode project is generated from a YAML spec by [XcodeGen](https://github.com/yonaskolb/XcodeGen) and the entire build/install/launch flow runs from the command line.

## Layout

```
iOS/
  README.md                    ← you are here
  project.yml                  XcodeGen spec — defines target, signing, dependencies, asset catalog
  HueHouseiOS.xcodeproj/       generated by `xcodegen generate` (regenerate after spec edits)
  HueHouseiOS/
    HueHouseiOSApp.swift       SwiftUI App entry point
    RootView.swift             TabView with Lights / Gradients / Bridge tabs
    LightsView.swift           Group picker + light list with on/off + brightness
    LightDetailView.swift      Per-light controls (brightness, color, presets)
    RGBAColorSheet.swift       Sheet-presented RGBA color picker
    GradientsView.swift        Built-in palettes + custom gradients + apply
    ImportGradientSheet.swift  CSS gradient / comma color list importer
    BridgeView.swift           Pairing flow + bridge status + appearance
    Info.plist                 NSLocalNetworkUsageDescription + NSBonjourServices declarations
    Assets.xcassets/
      AppIcon.appiconset/      iOS icon (regenerated by Scripts/generate-icon.swift)
```

All view code uses `import HueKit` to talk to the shared store, models, and bridge client — no AppKit anywhere in this directory. The repo's root `Package.swift` declares `HueKit` as a `.library` product on `[.macOS(.v14), .iOS(.v17)]`, so the iOS Xcode target consumes it as a local Swift package dependency.

## Prerequisites

- **Xcode 26+** (full Xcode app, not just Command Line Tools). If `xcodebuild -version` errors with *"requires Xcode"*, run `sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer` once, or prefix every `xcodebuild` call with `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer`.
- **XcodeGen** (`brew install xcodegen`) to generate `HueHouseiOS.xcodeproj` from `project.yml`.
- **An Apple ID signed into Xcode → Settings → Accounts.** The free Personal Team is enough for device testing (apps expire after 7 days). For App Store distribution you need the paid Apple Developer Program.

## First-time setup

1. **Find your Team ID.** Once your Apple ID is signed in and Xcode has issued a development certificate, run:
   ```sh
   security find-certificate -c "Apple Development" -p ~/Library/Keychains/login.keychain-db \
     | openssl x509 -text -noout | grep "Subject:"
   ```
   Look for the `OU=` field — that ten-character string is your Team ID. (The parenthetical inside the cert *Common Name* is **not** the team ID; it's a per-cert identifier.)
2. **Plug it into [project.yml](project.yml)** as `DEVELOPMENT_TEAM`.
3. **Generate the Xcode project:**
   ```sh
   cd iOS
   xcodegen generate
   ```

## Build, install, launch

The full deploy-to-device pipeline:

```sh
cd iOS

# 1. Build for the connected iPhone (xcodebuild auto-fetches a profile via -allowProvisioningUpdates).
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild -project HueHouseiOS.xcodeproj \
             -scheme HueHouseiOS \
             -destination 'id=<YOUR_DEVICE_UDID>' \
             -allowProvisioningUpdates \
             -derivedDataPath build-device \
             build

# 2. Install on the device.
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcrun devicectl device install app \
        --device <YOUR_DEVICE_UDID> \
        build-device/Build/Products/Debug-iphoneos/HueHouseiOS.app

# 3. Launch.
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcrun devicectl device process launch \
        --device <YOUR_DEVICE_UDID> \
        local.huehouse.ios
```

Find your device UDID with `xcrun devicectl list devices` (the `Identifier` column).

### First launch on the device

iOS won't run a Personal-Team-signed app until you trust the developer cert on the phone:

1. **Settings → General → VPN & Device Management → Developer App** on the iPhone.
2. Tap your developer cert (e.g. *Apple Development: jane@example.com*) → **Trust** → confirm.
3. Re-run step 3 above (`xcrun devicectl ... process launch`), or open the app from the home screen.

The trust persists per-cert — future installs don't repeat this dance. Free Personal Team apps stop launching after 7 days; rebuild and reinstall to refresh.

### Build for the simulator (no signing needed)

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild -project HueHouseiOS.xcodeproj -scheme HueHouseiOS \
             -destination 'generic/platform=iOS Simulator' \
             -derivedDataPath build CODE_SIGNING_ALLOWED=NO build
```

Then drag the resulting `.app` onto a booted simulator, or `xcrun simctl install booted <path>.app && xcrun simctl launch booted local.huehouse.ios`.

### Run HueKit tests on the iOS Simulator

The `HueKitTestsiOS` target is a logic-only unit-test bundle that re-runs the shared `Tests/HueKitTests/` Swift Testing suite against the iOS runtime. No host app is needed.

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild test -project HueHouseiOS.xcodeproj -scheme HueKitTestsiOS \
             -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
             -derivedDataPath build CODE_SIGNING_ALLOWED=NO
```

The same suite is also exercised on macOS via `swift test` from the repo root.

## App icon

The iOS icon is generated by the same script as the macOS icon, so they stay visually in sync:

```sh
swift Scripts/generate-icon.swift
```

The script writes the macOS `.icns` to `Packaging/AppIcon.icns` *and* a flat full-bleed iOS variant to `iOS/HueHouseiOS/Assets.xcassets/AppIcon.appiconset/AppIcon-1024.png`. iOS applies its own corner mask, so the iOS variant skips the macOS Squircle inset for a cleaner result on the home screen.

## Notes and known gaps

- iOS has no menu bar, no Dock — the macOS-only `MenuBarExtra` and the "Hide Dock icon" toggle do not exist here.
- The Liquid Glass material treatment from macOS isn't replicated; the iOS UI uses standard system materials and `.insetGrouped` Form / List styles instead.
- Local Network permission can get stuck in the "denied" state if the user dismisses the prompt without choosing. Reset via **Settings → Hue House → Local Network**.
